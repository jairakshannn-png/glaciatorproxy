Self-hosted web relay

Reach the page.
Leave your address behind.

GlaciatorProxy fetches pages on its own server and hands you the result, so the site you're visiting sees this proxy's connection instead of yours directly.

🔗

Try: wikipedia.org, or type anything to search DuckDuckGo

How a request travels

01

You enter an address

GlaciatorProxy normalizes it and treats bare search terms as a DuckDuckGo query.

02

The server fetches it

This server requests the page on your behalf and reads back the HTML, styles, and assets.

03

Links get rewritten

Every link, form, and asset URL on the page is rewritten to route back through the proxy, so browsing stays inside it.

What this actually protects — and what it doesn't

Hides your IP from the destination site

The site you visit sees this server's address, not yours.

Drops framing and CSP headers

Sites that try to block being loaded through a relay are overridden.

Not encryption or a VPN

Whoever runs the proxy server can see the pages you request. Only trust a server you control.

Your network still sees the proxy

Your ISP or local network can see that you connected to GlaciatorProxy, just not which pages you loaded through it.

Logins and heavy JS apps may break

Cookie-based sessions aren't rehomed to the proxy, and some single-page apps use techniques this rewriter doesn't catch.

GlaciatorProxy is a self-hosted tool. Run it on infrastructure you own or trust, and respect the terms of service of any site you access through it.
